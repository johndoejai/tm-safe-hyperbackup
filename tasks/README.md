# tasks/

Pro Aufgabe ein Unterordner mit zwei Dateien:

- `research.md` — was wir ueber den relevanten Code bzw. das System wissen,
  bevor wir aendern. Echte Beobachtungen, keine Annahmen.
- `plan.md` — konkreter Plan fuer die Aenderung, mit Dateipfaden, Code-Schnipseln
  und einer Todo-Liste, die waehrend der Umsetzung abgehakt wird.

Details zum Workflow stehen in `../CLAUDE.md` unter "Workflow".
